## 🚀 上传到 GitHub 的建议

### 方案一：只上传必需文件（推荐）
只上传根目录的文件，保持仓库简洁：
```bash
git add .
git commit -m "Initial commit: 青花瓷数字博物馆"
git push
```

### 方案二：上传所有文件（包括开发文件）
如果您想保留所有开发历史：
```bash
# 确保 .gitignore 配置正确
git add .
git commit -m "Initial commit: 青花瓷数字博物馆（包含开发文件）"
git push
```

### 方案三：排除备份文件夹
如果备份文件太大，可以在 `.gitignore` 中取消注释：
```
data/backup/
```

## 🎯 快速上传命令

```bash
# 1. 初始化 Git（如果还没有）
git init

# 2. 添加所有文件
git add .

# 3. 提交
git commit -m "Initial commit: 青花瓷数字博物馆"

# 4. 添加远程仓库（替换为您的仓库地址）
git remote add origin https://github.com/您的用户名/仓库名.git

# 5. 推送到 GitHub
git push -u origin main
```

或者使用提供的部署脚本：
```bash
./deploy.sh
```
